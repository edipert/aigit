# Socratic Gate Checklist

## Objective
Never start coding until you've successfully passed the Socratic Gate.

## Checklist for Complex Requests:
1. **Deep Discovery**: Have I asked 3 strategic, non-obvious questions about this feature?
2. **Edge Cases**: Ask what happens when the operation fails, or the inputs are malformed.
3. **Confirmation**: Have I obtained explicit user confirmation before writing the code? If yes, check if the architecture requires `{task-slug}.md`. If so, write the task first.

## Rule
If even 1% of the instruction is ambiguous, default to an interrogative response rather than an assumptive response. Wait for the user to confirm.
